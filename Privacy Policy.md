# BILITATO 隐私政策 (Privacy Policy)

最后更新：2026.5.14

BILITATO 是一款用于辅助分析 Bilibili 视频内容的浏览器扩展。本隐私政策说明本扩展会处理哪些数据、这些数据会被发送到哪里，以及用户如何管理相关数据。

---

## 1. 我们处理哪些数据

为了实现插件功能，BILITATO 可能会处理以下信息：

- 当前 Bilibili 视频页面中的字幕、标题、分段信息及相关页面内容
- 用户在插件中主动输入的问题、Prompt、自定义模型配置
- 用户填写的 API Key、模型名、Base URL 等设置项
- 为了生成无字幕视频转录结果而提取的音频内容
- 为了异常诊断而生成的匿名错误信息与运行环境信息，例如插件版本、浏览器环境、系统平台、错误类型、任务类型、BVID、是否存在字幕、字幕条数等
- 为了缓存和统计功能使用情况而生成的结果数据，例如：
  - 原始字幕
  - 处理后的字幕
  - 总结内容
  - 分段结果
  - 验真结果
  - 功能调用次数与时间

这些数据仅用于实现插件声明的功能，不用于广告投放，也不用于出售用户数据。

---

## 2. 数据会发送到哪里

### 2.1 用户选择的 AI 服务

当用户主动使用总结、分段、聊天、验真或音频转录功能时，BILITATO 可能会将实现该功能所必需的数据发送到用户选择或配置的第三方 AI 服务，包括但不限于：

- OpenAI
- Gemini
- DeepSeek
- 智谱 AI
- ModelScope
- Moonshot
- Groq
- 用户自行配置的兼容 API 服务

发送的数据可能包括：

- 视频字幕文本
- 视频标题
- 用户输入的问题
- 音频文件（仅在用户主动发起 ASR 转录时）
- 为了改善识别准确率而附带的上下文提示

这些请求直接从用户浏览器发送到相应服务提供商，并按照对应第三方服务的隐私政策处理。

### 2.2 Supabase 云端缓存与统计服务

BILITATO 当前使用 Supabase 提供以下功能：

- 云端缓存部分视频分析结果
- 记录功能调用统计

在相关功能启用时，插件可能会向 Supabase 写入或读取以下数据：

- 视频标题与 BVID
- 原始字幕 (`raw_subtitle`)
- 处理后的字幕 (`processed_subtitle`)
- 总结、分段、验真结果
- 字幕上传次数、上传时间
- 各功能调用次数
- 匿名使用统计（例如功能名、调用次数、Token 数、匿名用户标识）

这些数据用于：

- 避免重复生成相同视频的结果
- 提高缓存命中率
- 统计功能使用情况

插件不会将这些数据出售给第三方。

### 2.3 Sentry 异常诊断服务

BILITATO 使用 Sentry 进行匿名异常诊断，以帮助开发者发现和修复插件错误。异常诊断功能默认开启，用户可以在插件设置中通过“允许上报异常”选项关闭。

当插件发生程序异常、AI 请求异常、ASR 转录异常、云缓存异常、下载异常或其他非用户主动配置错误时，插件可能会向 Sentry 上报以下信息：

- 插件版本与 Manifest 版本
- 浏览器 userAgent、语言、操作系统与 CPU 架构
- 错误类型、错误信息、错误堆栈
- 当前任务类型，例如总结、分段、聊天、验真、ASR、下载
- AI 服务商标识，例如 OpenAI、DeepSeek、Gemini、Groq 等
- 页面类型，例如视频页或合集页
- 当前视频 BVID
- 是否存在字幕、字幕条数、是否启用 ASR

异常诊断不会上传以下内容：

- 用户 API Key、Groq API Key、Supabase 凭证或任何 token
- 完整字幕文本
- 用户聊天内容
- 用户自定义 Prompt
- 完整视频下载链接
- 自定义 API Base URL
- Cookie、账号信息、Bilibili 用户身份信息
- Sentry DSN

插件会在发送前对敏感字段进行过滤。未填写 API Key、当前视频无字幕、用户取消授权、自定义 API 地址未填写等用户配置或主动操作导致的提示，不会作为异常上报。

---

## 3. 本地存储的数据

BILITATO 会在用户浏览器本地存储以下信息：

- 用户设置与偏好
- 用户填写的 API Key、模型名、Base URL、自定义 Prompt
- 本地缓存的字幕、总结、分段、验真结果
- 匿名用户标识（用于统计功能调用，不包含真实身份信息）
- 是否允许上报异常的开关状态

这些数据通常保存在 Chrome Storage 中，由用户自行控制。

---

## 4. API Key 与凭证处理

- 用户填写的 API Key 默认保存在浏览器本地存储中
- API Key 不会发送给插件开发者
- API Key 仅用于向用户所选择的 AI 服务发起请求
- 开发者无法直接读取用户浏览器本地保存的 API Key

如果用户配置了自定义 API，插件还会在用户明确授权后请求访问该自定义域名的权限。

---

## 5. 权限与访问说明

BILITATO 申请的权限仅用于实现核心功能，例如：

- 访问 Bilibili 页面内容以读取字幕、标题和播放信息
- 下载用户主动选择的视频、音频和字幕文件
- 调用用户选择的 AI 服务
- 在用户主动启用自定义 API 时访问用户授权的自定义域名

对于自定义 API 域名访问，插件使用按域名单独授权的方式，不会默认访问所有站点。

---

## 6. 数据共享

BILITATO 不会出售用户数据。

但为了提供功能，插件可能会将必要数据共享给以下第三方服务：

- 用户主动选择的 AI 服务提供商
- 插件当前使用的 Supabase 云端缓存与统计服务
- Sentry 异常诊断服务

这些共享仅限于实现插件功能所必需的数据范围。

---

## 7. 用户如何控制和删除数据

用户可以通过以下方式管理数据：

- 在插件设置中修改或清空 API Key、模型和自定义配置
- 在插件设置中关闭“允许上报异常”，停止发送匿名异常诊断信息
- 清除浏览器扩展存储以删除本地缓存和设置
- 卸载扩展

对于已经发送到第三方 AI 服务、Supabase 或 Sentry 的数据，其后续处理受对应服务提供商的政策约束。

---

## 8. 第三方服务

如果用户使用第三方 AI 服务、触发云端缓存功能，或允许异常诊断功能，相关数据将由对应第三方服务根据其自身隐私政策处理。用户在使用前应自行了解这些服务的隐私政策和数据处理规则。

---

本隐私政策可能会不定期更新。如有重要变更，将在本页面更新最新版本。
